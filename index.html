Investigación de Vulnerabilidades Web

Inyección SQL · XSS · Fallos de Validación · Manejo Inseguro de Sesiones

1. Inyección a Base de Datos

La inyección SQL ocurre cuando una aplicación no valida o limpia correctamente los datos de entrada y permite que un atacante inserte comandos SQL maliciosos. Esto puede provocar filtración de datos, modificación o incluso eliminación de información crítica.

Para prevenirla, se recomienda:

  • Uso de consultas preparadas (prepared statements).
  • Validación y sanitización estricta de entradas.
  • Mínimos privilegios en cuentas de base de datos.

2. Cross-Site Scripting (XSS)

XSS permite que un atacante inyecte scripts maliciosos en páginas vistas por otros usuarios. Esto puede resultar en robo de cookies, redirecciones o ejecución de acciones en nombre del usuario.

Mitigaciones recomendadas:

  • Escapar todo contenido dinámico antes de mostrarlo en HTML.
  • Usar cabeceras como Content-Security-Policy (CSP).
  • Validar y sanitizar entradas de usuario.

3. Fallos de Validación

Ocurren cuando una aplicación no valida adecuadamente los datos recibidos. Esto permite que datos no esperados o malformados entren al sistema, generando errores de lógica o abriendo puertas a ataques más graves como XSS o SQLi.

Buenas prácticas:

  • Validar siempre del lado del servidor (no solo del cliente).
  • Definir formatos y reglas claras para cada campo.
  • Rechazar datos que no cumplan los parámetros definidos.

4. Manejo Inseguro de Sesiones

Cuando las sesiones no son gestionadas de forma segura, un atacante puede secuestrar sesiones, suplantar usuarios o mantener sesiones activas indebidamente.

Buenas prácticas:

  • Usar cookies seguras (Secure, HttpOnly, SameSite).
  • Renovar el ID de sesión tras iniciar sesión.
  • Establecer tiempos de expiración apropiados.
  • Evitar exponer tokens en URLs.
